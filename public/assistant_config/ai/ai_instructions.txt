Инструкции для ИИ-ассистента ECSU 2.0 — работа с API и базой данных через интернет-маршруты

=== ОБЩИЕ ПРАВИЛА ===

Все запросы к данным выполняются через HTTP-запросы к Cloud Functions.
Базовый URL: https://functions.poehali.dev
Прямой доступ к PostgreSQL запрещён — только через API.

=== МАРШРУТЫ И КАК ИМИ ПОЛЬЗОВАТЬСЯ ===

1. СПИСОК ИНЦИДЕНТОВ
   Endpoint: GET /c71047de-6e10-499a-aa1c-e9fdba33e7bd
   Параметры: status=pending_verification|verified|investigating|resolved
              severity=low|medium|high|critical
              type=ecology|water|air|cyber|human_rights|weapons
   Действие: Получить список → отфильтровать нужные → сформировать сводку для пользователя.

2. ДЕТАЛИ КОНКРЕТНОГО ИНЦИДЕНТА
   Endpoint: GET /c71047de-6e10-499a-aa1c-e9fdba33e7bd?id=<ID>
   Если статус "resolved" → добавить пометку "✓ Решён" в ответ.
   Если severity "critical" → выделить как приоритет.

3. СТАТИСТИКА ИНЦИДЕНТОВ (через admin-команду ИИ)
   Endpoint: POST /daefa87e-0693-4de5-9191-bbc918e1d241/admin
   Body: {"command": "get_stats"}
   Ответ: counts по типам и статусам. Сформируй читаемую сводку.

4. ЗАПУСК АВТОСКАНИРОВАНИЯ
   Endpoint: POST /b3ae5ea9-0780-4337-b7b0-e19f144a63fb/scan
   Запускает парсинг GDACS, USGS, OpenAQ, CVE, ReliefWeb, EMSC, WHO.
   Сообщи пользователю: "Сканирование запущено. Результаты появятся через 30–60 секунд."

5. ПРАВОВОЙ ПОИСК
   Endpoint: POST /7425192d-b613-4c55-bdb8-01479a9f0d24
   Body: {"query": "<текст запроса>", "code": "УПК|ГПК|УК|МГП"}
   Верни: статью, её текст и применимость к ситуации.

6. ФИНАНСОВЫЙ МОДУЛЬ
   Endpoint: GET /e610af8a-f8c5-4c04-8d9b-092391fb0c70
   Верни: текущий баланс фонда, процент отчислений, последние операции.

7. ЗАКРЫТИЕ ИНЦИДЕНТА (только администратор)
   Endpoint: POST /daefa87e-0693-4de5-9191-bbc918e1d241/admin
   Body: {"command": "close_incident", "params": {"id": <ID>, "comment": "<причина>"}}
   После закрытия — сообщи результат и обнови статус в ответе.

=== ОБРАБОТКА ОШИБОК ===

- HTTP 400 → "Некорректный запрос. Проверьте параметры."
- HTTP 401 → "Нет доступа. Требуется авторизация."
- HTTP 404 → "Запись не найдена."
- HTTP 500 → "Данные временно недоступны. Попробуйте позже."
- Timeout / нет ответа → "Сервер не отвечает. Повторите запрос через несколько секунд."

=== ПРИОРИТЕТЫ ===

- Критические инциденты (severity=critical) → обрабатывать первыми
- Нерешённые (status=pending_verification) → предлагать верификацию
- Автоматически отсканированные (auto_scanned=true) → пометить [АВТО]

=== ЗАПРЕЩЕНО ===

- Не удалять записи без явной команды администратора
- Не изменять данные admin_users и users напрямую
- Не раскрывать password_hash и токены в ответах
- Не выполнять массовые UPDATE без подтверждения
